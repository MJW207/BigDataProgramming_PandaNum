## 1. 프로젝트 개요

### 1.1 주제
**환경 통합 작물 질병 진행단계 분류 모델**

시설(비닐하우스)과 노지 환경을 통합하여, 국내 주요 작물의 질병 진행단계(정상/초기/중기/말기)를 분류하는 딥러닝 모델 개발.

### 1.2 차별점 (중간보고서 이후 재정의)

**차별점 1. 국내 데이터 기반**
- 기존 선행연구는 해외 데이터(PlantVillage 등) 및 해외 작물 중심
- AI Hub의 국내 작물 질병 진단 데이터를 기반으로 국내 농업 환경에 최적화

**차별점 2. 시설·노지 통합 환경**
- 기존 연구는 시설 또는 노지 중 한쪽 환경에만 학습하여 도메인 편향 발생
- 시설·노지 양측에서 일관된 성능을 보이는 범용 분류 모델 설계

**차별점 3. 진행단계 분류 + 개방형 일반화**
- 기존 국내 연구는 질병 종류(병명) 분류에 집중, 진행단계 분류 연구 부족
- 질병 진행에 따른 공통 시각적 특징을 학습하여, **학습에 없던 새로운 작물 종류에도 일정 수준 진행단계를 추정할 수 있는 개방형 일반화** 가능성 확보

## 3. 데이터 전처리 파이프라인

### 3.1 설계 방침

원본 이미지(~1920×1820 이상)에서 바로 전처리. 중간 512px 저장 단계 제거.

```
원본 이미지 (1920×1820~3024×3024)
    ↓
1. Bounding Box 크롭 (JSON points 활용)
    ↓
2. 30% 패딩 추가 (병변 주변 맥락 보존)
    ↓
3. Letterbox Resize → 모델 입력 크기 (종횡비 유지, INTER_LANCZOS4)
    ↓
4. 채널 정규화 (데이터셋 통계 기반 mean/std)
    ↓
5. 학습용 증강 (RandomHorizontalFlip, RandomRotation 등)
```

### 3.2 이미지 해상도 및 종횡비

| 환경 | 원본 해상도 | 종횡비 |
|---|---|---|
| 시설 | 3024 × 3024 | 1:1 (정사각형) |
| 노지 | 3024 × 4032 | 3:4 (세로 직사각형) |

두 환경 간 해상도가 다르기 때문에 단순 resize 시 왜곡 발생. Letterbox resize(종횡비 유지)가 필수인 근거.

### 3.3 Bounding Box 크롭 상세

```
bbox 좌표 (원본 기준): xtl, ytl, xbr, ybr
원본 크기: description.height, description.width

→ 원본에서 직접 크롭 (스케일 변환 불필요)
→ 패딩: w × 0.3, h × 0.3 (이미지 경계 클램핑)
→ 최소 크기 보장: 크롭 결과가 너무 작으면 원본 이미지 사용
→ 시설(1:1)·노지(3:4) 크롭 결과 모두 Letterbox로 통일
```

### 3.4 EfficientNet 버전별 입력 크기

| 버전 | 입력 크기 | 원본 대비 압축 | 선택 여부 |
|---|---|---|---|
| B0 | 224×224 | 기준 모델 | ✓ 비교용 |
| B1 | 240×240 | — | — |
| B2 | 260×260 | — | — |
| **B3** | **300×300** | **병변 디테일 보존↑** | **✓ 핵심 실험** |
| **B4** | **380×380** | **고해상도** | **✓ 시간 여유 시** |
| B5+ | 456px~ | 연산량 과다 | — |

### 3.5 클래스 불균형 처리 전략

#### 핵심 원칙
각 작물×환경 그룹별로 불균형 패턴이 완전히 다름. 하드 언더샘플링이나 단순 소프트 캡은 적합하지 않음.

```
시설 딸기:   정상 19,679 >> 말기 3,795  → 정상이 압도
시설 단호박: 초기  6,358 >  정상 2,751  → 정상이 가장 적음
노지 고추:   말기 10,901 >  초기 2,849  → 말기가 가장 많음
시설 상추:   전 클래스 10,161~11,476    → 거의 균형
```

#### 해결: 그룹×클래스 단위 역빈도 가중치 자동 계산

```python
# 샘플 i의 WeightedRandomSampler 가중치:
weight_i = 1.0 / (전체_그룹_수 × 해당_그룹_내_클래스_수)
```

- 그룹 수 = 12 (학습 작물×환경 조합)
- 해당_그룹_내_클래스_수 = 그 샘플이 속한 (env, crop, risk) 조합의 샘플 수
- 어떤 분포 패턴에도 자동으로 균형 달성 — 수동 설정 불필요

**효과:**
- 딸기 정상(19,679) → 낮은 가중치 자동 부여
- 단호박 정상(2,751) → 상대적으로 높은 가중치 자동 부여
- 노지 고추 말기(10,901) → 낮은 가중치 자동 부여
- 12개 그룹이 학습에 동등하게 기여

```python
# num_samples로 에폭당 샘플 수 조절 (학습 시간 제어)
WeightedRandomSampler(weights=w, num_samples=40_000, replacement=True)
```

#### 잔여 불균형 처리
- **Focal Loss**: 가중치 샘플링 후 남은 어려운 샘플(hard example) 집중
- 두 메커니즘이 상호 보완적으로 작동

### 3.6 데이터 분할

```
전체 학습 데이터 (12그룹) → 70:15:15 Stratified Split
    train (70%) → 모델 학습
    val   (15%) → 에폭별 성능 모니터링, Early Stopping
    test  (15%) → 최종 성능 평가

held-out 데이터 (6그룹) → 별도 보관, 학습 완료 후 개방형 일반화 평가
```

## 4. 모델 설계

### 4.1 학습 파이프라인 구조

2단계로 구성. 피처 추출 단계에서 사전학습 가중치의 특성 추출 능력을 기준점으로 확인하고, Fine-tuning 단계에서 작물 도메인에 특화된 학습 수행.

```
[Stage 1 - Feature Extraction]
EfficientNet-Bx (백본 완전 동결) → CBAM → Pool → Flatten
    → MLP 분류기 (Linear → BN → ReLU → Linear(4)) 만 학습
    → 목적: ImageNet 사전학습 가중치의 병변 특징 추출 능력 기준점 확인

[Stage 2 - Fine-tuning]
EfficientNet-Bx + CBAM 전체 학습 (2단계 점진적 Unfreeze)
    → Phase 1: 백본 동결, CBAM + FC만 학습 (3~5 에폭)
    → Phase 2: 전체 Unfreeze, 레이어별 차등 학습률
    → 목적: 작물 질병 도메인 특화 학습
```

### 4.2 모델 구조 상세

```
EfficientNet-Bx (ImageNet 사전학습)
    → CBAM (채널 어텐션 + 공간 어텐션)
    → AdaptiveAvgPool2d(1) → Flatten
    → Dropout(0.5) → Linear(d→512) → BN → ReLU → Dropout → Linear(512→4)
    → 출력: 정상(0) / 초기(1) / 중기(2) / 말기(3)
```

**버전별 Flatten 출력 차원 (d):**

| EfficientNet | Flatten 출력 차원 |
|---|---|
| B0 | 1,280 |
| B3 | 1,536 |
| B4 | 1,792 |

### 4.3 CBAM 선택 이유

- 병변이 이미지의 일부 영역에만 국소적으로 나타남
- 배경(흙, 줄기, 잎 전체) 면적이 커서 모델이 배경에 주목할 위험
- 채널 어텐션: 병변 관련 채널(색상, 텍스처) 강조
- 공간 어텐션: 병변 위치 집중
- Grad-CAM으로 어텐션 효과 시각적 검증 예정

### 4.4 2단계 Fine-tuning 학습률 전략

**Phase 1 (3~5 에폭):** 백본 동결, CBAM + FC Layer만 학습
- ImageNet 사전학습 가중치 보존
- 새로 추가된 레이어 먼저 안정화

**Phase 2 (나머지):** 전체 Unfreeze, 레이어별 차등 학습률
```
features 하위층 × 0.1   (범용 시각 특징 보존)
features 상위층 × 0.2   (작물 도메인 적응)
CBAM         × 0.5   (병변 어텐션 적응)
FC Layer     × 1.0   (분류기 적극 갱신)
```

### 4.5 학습 설정

| 항목 | 설정값 | 비고 |
|---|---|---|
| 손실함수 | Focal Loss | γ 값 튜닝 대상, 클래스 불균형 보완 |
| 옵티마이저 | AdamW | Weight Decay로 과적합 억제 |
| 스케줄러 | CosineAnnealingWarmRestarts | 주기적 학습률 리셋으로 정체 구간 탈출 |
| 조기 종료 | patience=10 | Val Loss 기준 |
| 배치 크기 | 32 or 64 | GPU 메모리에 따라 조정 |
| Dropout | 0.3~0.5 | 튜닝 대상 |
| 에폭당 샘플 수 | num_samples 설정 | WeightedRandomSampler 제어 |

### 4.6 Soft Labeling (추가 기법)

중간보고서에서 중기↔초기 혼동이 32.7%로 가장 심각했던 문제를 완화하기 위해 도입 검토.

**도입 배경:**
- 진행단계는 연속적 스펙트럼 (정상→초기→중기→말기)
- 표준 One-hot 라벨은 인접 단계 간 유사성을 반영하지 못함
- 중기 이미지를 초기로 오분류하는 것과 말기로 오분류하는 것의 페널티가 동일 → 부적절

**방식:**
```
기존 One-hot:  중기 = [0.00, 0.00, 1.00, 0.00]
Soft Label:    중기 = [0.00, 0.10, 0.80, 0.10]  ← 인접 단계에 일부 확률 분산
```

- 인접 단계(초기·말기)에 α 만큼 확률 분산, 비인접(정상)은 0 유지
- α 값은 튜닝 대상 (0.05~0.15 범위)
- 구현: 커스텀 손실함수로 표준 Focal Loss 대체 또는 병행

**적용 시점:** Exp-3 (B3) 이후 성능이 충분하지 않을 경우 추가 실험으로 수행.

### 4.7 평가 지표

| 지표 | 역할 |
|---|---|
| **Macro F1-Score** | 핵심 지표 — 클래스 불균형 무관하게 전 클래스 동등 반영 |
| Accuracy | 보조 참고 |
| 클래스별 Precision / Recall | 초기·말기 등 소수 클래스 세부 분석 |
| Confusion Matrix | 인접 단계 간 오분류 패턴 파악 |
| Grad-CAM | CBAM 병변 집중 효과 시각적 검증 |

## 5. 실험 설계

### 5.1 파일럿 결과 및 Baseline 정의

#### 파일럿 결과 (참고용, 비교 기준 아님)

중간보고서 이후 Runpod에서 수행한 결과. 실험 데이터 확인 결과 **시설 토마토 단일 작물**로만 학습된 것으로 확인됨.

| 항목 | 값 |
|---|---|
| 모델 | EfficientNet-B0 + CBAM |
| 실제 학습 데이터 | 시설 토마토 단일 (증강 포함 34,506장) |
| bbox 크롭 | 미적용 |
| Best Val Macro F1 | 0.8729 |
| 용도 | 파이프라인 동작 확인용 — 단일 작물이므로 비교 기준으로 부적합 |

#### 실제 Baseline (Exp-1)

12그룹 통합 데이터로 첫 실험을 수행하여 그 결과를 이후 모든 실험의 비교 기준으로 삼음.

### 5.2 메인 실험: 변수 분리 비교

12그룹 통합 학습 기반으로, 변수를 하나씩 추가하며 각 요소의 효과를 독립적으로 측정.

| 실험 ID | EfficientNet | bbox crop | Soft Label | 학습 데이터 | 측정 목적 |
|---|---|---|---|---|---|
| **Exp-1** | B0 | ✗ | ✗ | 12그룹 통합 | **실제 Baseline** — 새 데이터 구성 기준점 |
| **Exp-2** | B0 | ✓ | ✗ | 12그룹 통합 | bbox 크롭 효과 (Exp-1 대비) |
| **Exp-3** | **B3** | ✓ | ✗ | 12그룹 통합 | 해상도 향상 효과 (Exp-2 대비) — 핵심 모델 |
| Exp-4 | B4 | ✓ | ✗ | 12그룹 통합 | 고해상도 상한 확인 (선택) |
| Exp-5 | B3 | ✓ | ✓ | 12그룹 통합 | Soft Labeling 효과 (Exp-3 대비, 선택) |

> Exp-1→2→3 순서로 변수를 하나씩 추가해 각 요소의 기여도 분리 측정.  
> Exp-3이 핵심 모델. Exp-4·5는 추가 개선이 필요할 때만 수행.

### 5.3 추가 비교: Feature Extraction vs Fine-tuning

Exp-1 학습 시에만 수행. 노트북에 이미 구현된 파이프라인 재활용.

| 베이스라인 | 방식 | 목적 |
|---|---|---|
| BL-1 (Feature Extraction) | 백본 완전 동결, MLP만 학습 | 사전학습 가중치 원본 성능 기준점 |
| BL-2 (Fine-tuning, 제안) | 2단계 점진적 Unfreeze | 도메인 적응 효과 측정 |

> Exp-2~5에서는 BL-2(Fine-tuning)만 수행. BL-1 비교는 Exp-1에서 한 번만 수행하여 Fine-tuning 우위를 새 데이터 기준으로 재확인.

### 5.4 차별점 2 검증: 환경 일반화 분석

**별도 학습 없이 Exp-3(핵심 모델) 결과를 활용한 분석.** 통합 학습 모델이 두 환경에서 일관된 성능을 보이는지 검증.

#### 분석 1: 시설/노지 분리 평가

```
Exp-3 통합 학습 모델 → 시설 test split 성능 vs 노지 test split 성능 비교

측정 지표:
  - 시설 Macro F1 vs 노지 Macro F1 (차이가 작을수록 환경 일반화 성공)
  - 클래스별 F1 환경 간 비교 (특정 클래스만 편중되는지 확인)
  - 시설/노지 각각의 Confusion Matrix
```

#### 분석 2: 고추 Cross-Environment 분석

학습 데이터 중 시설·노지 양쪽에 존재하는 유일한 작물인 **고추**를 대상으로 환경 일반화 시각적 검증.

```
대상: 시설 고추(crop_code=2) + 노지 고추(crop_code=1)

분석 내용:
  - 시설 고추 test 성능 vs 노지 고추 test 성능 비교
  - Grad-CAM 시각화: 두 환경 모두에서 병변 영역에 집중하는지 확인
  - 환경별 시각 패턴 차이에도 모델이 같은 병변 특징을 잡아내는지 검증
```

> 시설 애호박(말기 0장), 시설 오이(min=176)는 학습 제외되어 양환경 학습 작물은 고추 하나만 가능.

### 5.5 차별점 3 검증: 개방형 일반화 실험

Exp-3(핵심 모델)으로 수행. 학습에 사용하지 않은 작물에서도 진행단계를 추정 가능한지 정량 검증.

```
대상: held-out 6그룹 (학습에 미사용)
  시설: 가지, 참외, 수박
  노지: 호박, 양배추, 잎마름병(토마토)

측정:
  - 학습 내 test split Macro F1 vs held-out Macro F1 비교
  - 그룹별 (작물×환경) Macro F1 개별 분석
  - Grad-CAM으로 본 적 없는 작물에서도 병변에 집중하는지 시각 검증
  - 클래스별 Confusion Matrix로 어느 단계에서 일반화가 어려운지 분석
```

### 5.6 실험 우선순위

최종 발표(2026-06-10)까지 일정 고려한 우선순위.

| 분류 | 항목 | 차별점 검증 |
|---|---|---|
| **필수** | Exp-1, Exp-2, Exp-3 | 핵심 모델 결정 |
| **필수** | 시설/노지 분리 평가 (Exp-3 결과 활용) | 차별점 2 |
| **필수** | 개방형 일반화 (held-out 테스트) | 차별점 3 |
| 중요 | 고추 Cross-Environment 분석 (Grad-CAM) | 차별점 2 보강 |
| 중요 | BL-1 vs BL-2 (Exp-1 내) | 모델 선정 근거 |
| 선택 | Exp-4 (B4) | 해상도 상한 |
| 선택 | Exp-5 (Soft Labeling) | 중기↔초기 혼동 완화 |

**시간 부족 시 처리:**
- 선택 항목 → 최종 보고서 "한계점 및 향후 연구"에 언급
- 중요 항목 → 정량 결과 없이 정성 분석만 기술

### 5.7 가설 및 검증 방법

| 가설 | 검증 실험 |
|---|---|
| 작물 다양화(4종→12그룹) + 가중치 균형이 안정적 학습을 가능하게 한다 | Exp-1 (Baseline 확립) |
| bbox 크롭이 전체 이미지 입력보다 성능 높다 | Exp-2 vs Exp-1 |
| B3이 B0보다 병변 디테일 보존으로 성능 높다 | Exp-3 vs Exp-2 |
| Fine-tuning이 Feature Extraction보다 성능 높다 | BL-2 vs BL-1 (Exp-1 내) |
| 통합 학습 모델이 시설·노지 양쪽에서 일관된 성능을 보인다 | 시설/노지 분리 평가 |
| 학습에 없던 작물에서도 진행단계 추정 가능하다 | 개방형 일반화 실험 |
| Soft Labeling이 인접 단계 혼동을 감소시킨다 | Exp-5 vs Exp-3 (선택) |
